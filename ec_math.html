Introduction to Finite Fields
  • \(\mathbb{F}_{p} = \left\{0,1,2,...,p-1\right\}\)

  • Examples:
    • \(\mathbb{F}_{7} = \left\{0,1,2,3,4,5,6\right\}\)
    • \(\mathbb{F}_{21} = \left\{0,1,2,3,...,19,20\right\}\)
    • \(\mathbb{F}_{127} = \left\{0,1,2,3,...,125,126\right\}\)
  • Finite field \(\mathbb{F}_{p}\) is simply a set of integers from \(0\) to \(p-1\).
  • Next, we can define arithmetic operations over finite fields.
    (addition/substraction, exponentiation, multiplication, division)
Finite Field Addition:
\(5+8=1\:\left(\bmod 12\:\right)\)
  • Imagine all numbers of finite field \(\mathbb{F}_{12}\) wrapped around a circle, like a clock.
  • For every addition operation, move the needle n steps clockwise.
  • This is equivalent to taking the remainder if the division: (5 + 8)/12 = 13/12 = 1 Remainder 1
Finite Field Subtraction:
\(4 - 19 = 2 \:\left(\bmod 17\:\right)\)
  • For a subtractive operation, simply move the needle n steps counter-clockwise.
  • In our example above, we first move 4 ticks clockwise (positive) then 19 ticks counter-clockwise (negative) to arrive at 2.
Finite Field Multiplication:
\(4 \cdot 5 = 3 \:\left(\bmod 17\:\right)\)
  • \(4 \cdot 5 = +20\), so we simply move the needle by +20 ticks clockwise and arrive at 3.
Finite Field Exponentiation:
\(2^{5}=15\:\left(\bmod 17\:\right)\)
  • \(2^{5}=32\), so we move the needle by +32 ticks clockwise and arrive at 15.
Finite Field Division:
Division
Multiplication Table
  • \(2/4=2\:\left(\bmod 6\:\right)\)
  • \(3/5=4\:\left(\bmod 17\:\right)\)
  • \(2=4 \cdot 2\:\left(\bmod 6\:\right)\)
  • \(3=5 \cdot 4\:\left(\bmod 17\:\right)\)
Generally, division over \(\mathbb{F}_{p}\) can only be solved by maintaining
a multiplication table for all element pairs of \(\mathbb{F}_{p}\).
  • Division can simply be defined as the inverse of multiplication.
  • However, computing modular division this way requires the lookup of smultiplication tables.
  • Yet, for prime field \(\mathbb{F}_{p}\), where P is prime, there is a direct way to compute finite field division ...
Division over over Prime Fields:
Fermat's little theorem
For prime field \(\mathbb{F}_{p}\)
  • Where p is prime:
  • \(n^{p-1} = 1\left(\bmod p\:\right)\)
  • \(\frac{1}{n} = n^{-1}\left(\bmod p\:\right)\)
  • \(\frac{1}{n} = n^{-1}\cdot n^{p-1} = n^{p-2} \left(\bmod p\:\right)\)
Since this is equal to one, we can apply to the division equation:                                           
  • Fermat’s little theorem is valid for all prime fields \(\mathbb{F}_{p}\), where p is prime.
  • We use Fermat’s little theorem to derive division for prime fields.
  • To compute prime field division: \(\frac{1}{n} = n^{p-2} \left(\bmod p\:\right)\)

Ellipic Curves over Real Numbers:
Curve shown: \(y^{2}=x^{3}+7\)
  • Elliptic curve general form:
  • \(y^{2}=ax^{3}+bx+c\)

  • The secp256k1 curve form:
  • \(y^{2}=x^{3}+7\)

  • Consider all points on the elliptic curve (EC) as EC-points (x,y).

  • Due to the shape of elliptic curves, we can next define EC point arithmetic...
EC-Point Addition over Real Numbers:
  • 1) Form a line with P1 & P2
  • 2) Intersect resulting line with EC
  • 3) Reflect intersection point across X-axis for P3
  • Properties of EC addition:
  • Commutative*: P1 + P2 = P2 + P1
  • Associative* : (P1 + P2) + P3 = P1 + (P2 + P3)
  • *Commutativity, Associativity proofs not provided here.
Computing EC-Point Addition over Real Nr's
  • For an elliptic curve of form:
    • \(y^{2}=ax^{3}+bx+c\)

  • The computation of \(P1 + P2\)
    • \(P_{3}(x_{3},y_{3}) = P_{1}(x_{1},y_{1}) + P_{2}(x_{2},y_{2})\)
    • \(s = \frac{y_{2}-y_{1}}{x_{2}-x_{1}}\) for \(x_{1} \neq x_{2}\)
    • \(x_{3} = s^{2}-x_{1}-x_{2}\)
  • The equations shown above which describe EC-point addition can be derived from the general elliptic curve \(y^{2}=ax^{3}+bx+c\) form.
  • However these are only valid for P1 + P2 where \(x_{1} \neq x_{2}\).
EC-Point Addition where P1=P2:
\(P_{3} = P_{1} + P_{1} = 2 \cdot P_{1}\)
  • Adding a point to itself:
  • Consider Line L'(P1,P2) as we move P1 & P2 closer: Line L eventually becomes the tangent of P1 (L'''), as P1 gets closer to P2.
  • Therefore, we can add a point P1 to itself, by taking the tangent at point P1 to intersect with the EC Curve.

  • \(P_{3}(x_{3},y_{3}) = 2 \cdot P_{1}(x_{1},y_{1})\)
  • \(s =\frac{(3x_{1}^{2}+a)}{2y_{1}}\)
  • \(x_{3} =s^{2}-2x_{1}\)
  • \(y_{3} =s(x_{1}-x_{3})-y_{1}\)
EC Addition with Infinity:
\( P_{1}(x_{1},y_{1}) + (Inf/Inf) = P_{1}(x_{1},y_{1}) \)
  • The Infinity Point:
  • The (Inf/Inf) point is defined as a point which is infinitely far away in the direction of the y-axis.

  • Therefore, we can add a point P1 to the infinity point simply by connecting a vertical line through P1.

  • \( P_{1}(x_{1},y_{1}) + (Inf/Inf) = P_{1}(x_{1},y_{1}) \)
  • \( P_{1}(x_{1},y_{1}) + P_{2}(x_{1},-y_{1}) = (Inf/Inf) \)

EC-Point Addition over \( \mathbb{F}_{p} \):
  • EC-Point Addition over
    Real Numbers \(\mathbb{R}\):
  • For x1 != x2:
  • \( s = \frac{y_{2}-y_{1}}{x_{2}-x_{1}} \)
  • \( x_{3} = s^{2}-x_{1}-x_{2} \)
  • \( y_{3} = s\:(x_{1}-x_{3})-y_{1} \)
  • Addition for P1 = P2:
  • \( s =\frac{(3x_{1}^{2}+a)}{2y_{1}} \)
  • \( x_{3} =s^{2}-2x_{1} \)
  • \( y_{3} =s(x_{1}-x_{3})-y_{1} \)
  • Addition with infinity:
  • \( (x_{1},y_{1}) + (\infty , \infty) = (x_{1},-y_{1}) \)
  • EC-Point Addition over Prime Field \( \mathbb{F}_{p} \) :

  • All EC point addition over \( \mathbb{R} \) equations still apply to EC point addition over \( \mathbb{F}_{p} \).

  • All previously derived EC equations over \( \mathbb{R} \) all apply to finite fields. We simply apply finite field arithmetic to the EC-point addition equations.

EC-Point Point Groups:
\( y^{2}=x^{3}+7\) over \( \mathbb{F}_{29} \)
  • We can generate a point group using EC curve point addition and a chosen generator point on the EC curve, G.
    • G
    • 2G = G + G
    • 3G = G + 2G = G + G + G
    • ...
    • nG = (Inf/Inf)

  • Group Order: We eventually arrive at the infinitiy point at the n'th member of the group. The points themselves form a finite point group of the order n:
    • { G, 2G, 3G, ..., nG }

  • For Example, we may choose (6,7) as our generator point:
    • { G(6,7), 2G(12,13), 3G(12,16}, 4G{6, 22}, 5G(inf,inf) }
Bitcoin Private & Public Keys
  • The secp256k1 EC point group:
  • \(y^{2}=x^{3}+7\) over \(\mathbb{F}_{p}\) where \(p = 2^{256} - 2^{32} - 977\)
  • \(G = \:(\) 79BE667E F9DCBBAC 55A06295 CE870B07 029BFCDB 2DCE28D9 59F2815B 16F81798,
    \(\:\:\:\:\:\:\:\:\:\:\:\:\) 483ADA77 26A3C465 5DA4FBFC 0E1108A8 FD17B448 A6855419 9C47D08F FB10D4B8    \(\:)\)
  • \(n =\) FFFFFFFF FFFFFFFF FFFFFFFF FFFFFFFE BAAEDCE6 AF48A03B BFD25E8C D0364141
  • Bitcoin private & public keys:
  • \(P = s \cdot G\)
  • (Private key scalar \(s\) is chosen, secp256k1-point \(P\) is the public key)
  • The secp256k1 EC-Point Group is used to generate Bitcoin private/public keys.
Public Key / PointSerialisation:
  • Uncompressed serialisation:
  • 04-Prefix (1byte) + X-coordinate (32bytes) + Y-coordinate (32bytes)
  • 0428026f91e1c97db3f6453262484ef5f69f71d89474f10926aae24d3c3eeb5f00 c41b6810b8b305a05de2b4448d7e2a079771d4c018b923a9ab860e4b0b4f86f6
  • Compressed serialisation:
  • 02/03-Prefix (1byte) + X-coordinate (32bytes)
  • 0228026f91e1c97db3f6453262484ef5f69f71d89474f10926aae24d3c3eeb5f00
  • In compressed form, the y-coordinate is omitted, since it can be derived from \(y^{2}=x^{3}+7\). However, there are two possible y values for each valid x-coordinate.
  • Therefore, the x coordinate is preceded by a even (02) or odd (03) prefix, indicating whether the even or odd y-coordinate is implied.
Libbitcoin:EC Math Classes & Methods
  • Libbitcoin provides byte_array classes for scalar and ec-point objects...
    • bc::ec_secret
    • bc::ec_uncompressed, bc::ec_compressed

  • And methods to generate points from s * P curve multiplication.
    • bc::ec_multiply()
    • bc::secret_to_public()
// 32 Byte Scalar = EC secret.
typedef byte_array<ec_secret_size> ec_secret;

// EC point uncompressed.
// Uncompressed size is 65 bytes.
typedef byte_array<ec_uncompressed_size> ec_uncompressed;

// EC point compressed.
// Compressed size is 33 bytes.
typedef byte_array<ec_compressed_size> ec_compressed;

// EC point *= secret multiplication.
bool ec_multiply(ec_compressed& point, const ec_secret& secret);
bool ec_multiply(ec_uncompressed& point, const ec_secret& secret);

// Generate public key from secret, using generator point G.
bool secret_to_public(ec_compressed& out, const ec_secret& secret);
bool secret_to_public(ec_uncompressed& out, const ec_secret& secret);

// Verify secret or point is valid (secp256k1).
bool verify(const ec_secret& secret);
bool verify(const ec_compressed& point);
bool verify(const ec_uncompressed& point);
Libbitcoin: EC Math Example
  • In this code example, we will generate Bitcoin private and public keys from scratch.

  • Both can be created from hex string literals, or generated from a source of entropy.

  • Note that both secret and point classes are stored as fixed-size byte arrays, and that some Libbitcoin methods require a conversion to or from byte vectors (data chunk).
    • bc::to_chunk()
    • bc::to_array<array_size>()
#include <bitcoin/bitcoin.hpp>
#include <iostream>

int main() {

  // Create scalar & uncompressed secp256k1 point from hex string.
  // ---------------------------------------------------------------------------

  // Create a 32 byte scalar object from a  hex string.
  bc::ec_secret my_scalar = bc::base16_literal(
    "28026f91e1c97db3f6453262484ef5f69f71d89474f10926aae24d3c3eeb5f00");

  // Create an 65 byte uncompressed point from hex string.
  bc::ec_uncompressed my_uncompressed_point = bc::base16_literal(
    "04" // Prefix
    "28026f91e1c97db3f6453262484ef5f69f71d89474f10926aae24d3c3eeb5f00"
    "c41b6810b8b305a05de2b4448d7e2a079771d4c018b923a9ab860e4b0b4f86f6");


  // Create a random private key and derive the public key
  // ---------------------------------------------------------------------------

  bc::data_chunk my_entropy(bc::ec_secret_size); // Secret size: 32bytes.
  bc::pseudo_random_fill(my_entropy);

  // Convert data chunk (byte vector) to ec secret (byte array).
  bc::ec_secret my_random_secret = bc::to_array<bc::ec_secret_size>(my_entropy);

  // Verify that secret is secp256k1.
  if( bc::verify(my_random_secret) )
  {

    bc::ec_compressed generator_point = bc::base16_literal(
      "02"
      "79be667ef9dcbbac55a06295Ce870b07029bfcdb2dce28d959f2815b16f81798");

    // Multiply with secp256k1 generator point.
    bc::ec_multiply(generator_point, my_random_secret);
    bc::ec_compressed my_public_key(generator_point);

    // Verify point is secp256k1.
    std::cout << bc::verify(my_public_key) << std::endl;

    // Print out public key.
    // Convert to data chunk (byte vector) and convert to hexstring.
    std::cout << bc::encode_base16(bc::to_chunk(my_public_key)) << std::endl;

    // Better: Direct deriviation secret -> public key.
    bc::ec_compressed my_public_key_2;
    bc::secret_to_public(my_public_key_2, my_random_secret);
    std::cout << (my_public_key == my_public_key_2) << std::endl;
  }

  else
  {
    std::cout << "Invalid secret chosen" << std::endl;
  }

  return 0;

}
